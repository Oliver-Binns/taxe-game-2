Getting Started

Introduction

To start using our code you will need to install

  • The Java JDK
  • Eclipse
  • Eclipse Gradle Plugin

The actual version of Eclipse you end up using shouldn't matter too much however this guide was built with Luna in mind. You could also use another IDE such as IntelliJ however the setup for that is not covered here.

You will also need to install the GitHub desktop client (we found this the easiest way to use Git but you can use other methods too).

Other Problems

If you have any problems getting set up. Please double check that you have read the full start up guide carefully. If that does not work please consult the FAQ.

Installing the Java JDK

To install the Java JDK you will need to download version 7 here. Accept the license agreement and download the correct file for your machine in the Java SE Development Kit 7u71 section.

Windows

  1. Wait for the .exe file to download and run it to install the JDK.
  2. After installing the JDK you will need to add a new Environment variable.
  3. Go to Control Panel -> System and Security -> System.
  4. Select Advanced system settings -> Environment Variables.
  5. You will then need to find the "Path" variable in the System Variables section.
  6. Double click to edit the value.
  7. You will now need to find the location the installer saved the JDK to. In my case this will be: C:\Program Files\Java\jdk1.7.0_71\bin. However, double check your Program Files folder to see if this matches.
  8. Add ';' plus the path name to the end of the variable value. So in my case ";C:\Program Files\Java\jdk1.7.0_71\bin" is appended to the end of the Path.

Mac

Double click the .dmg file to lauch it and install the JDK.

Installing Eclipse and Plugins

Install Eclipse

To download Eclipse 4.4 go to their downloads page here and download the appropriate files for your machine under the "Eclipse IDE for Java Developers" section. Even if you have a version of Eclipse you are best off downloading the latest update for everything to run smoothly.

Windows

  1. After downloading your zip file extract it to your preferred location (I recommend under C:. You can sometimes have problems if you try and put it under C:\Program Files or C:\Program Files (x86) when it comes to installing Plugins).
  2. Create a Desktop Shortcut by right clicking eclipse.exe -> Send to -> Desktop (create shortcut).
  3. Double click the Desktop shortcut to launch eclipse.
  4. Set your preferred Workspace folder

Mac

Install Plugins

All of these Plugins must be installed to run and import the project correctly. If you do not see an option to Install New Software under the Help drop down in Eclipse you may need to move your "eclipse" installation folder out of the C:\Program Files or C:\Program Files (x86) folder.

Gradle

  1. Open Eclipse
  2. Go to Help -> Install New Software...
  3. Paste "http://dist.springsource.com/release/TOOLS/gradle" into the "Work With:" text box.
  4. Select the drop down next to "Extensions / Gradle Integration" and tick the box next to "Gradle IDE".
  5. Press Next a few times and accept the license to install.

Android ADT-plugin

  1. Open Eclipse
  2. Go to Help -> Install New Software...
  3. Paste "https://dl-ssl.google.com/android/eclipse/" into the "Work With:" text box.
  4. Select the tick box next to "Developer Tools".
  5. Press Next a few times and accept the license to install.

GWT Plugin

  1. Open Eclipse
  2. Go to Help -> Install New Software...
  3. Paste "https://dl.google.com/eclipse/plugin/4.4" into the "Work With:" text box. (Again this is assuming you have the Eclipse version 4.4.).
  4. Select the drop down next to "SDK's" and select "Google Web Toolkid SDK 2.6.0"
  5. Press Next a few times and accept the license to install.

Importing the Project

Finally you are all set up to import the latest version of the project. To do this we will use the GitHub desktop client which can be downloaded for Windows or Mac if you are using Linux you will have to use traditional Git commands as described here. You will also need to login with the GitHub account which has permission to access the project.

  1. After installing the client the first thing to do is to make a clone of the project. To do this first select the '+' icon in the top left corner. Then when the drop down box appears select Clone and select LocommotionCommotion. If LocommotionCommotion or the Clone option itself does not appear contact Callum Hewitt at ch1194@york.ac.uk.

  2. You will then have the option to specify a directory to store the clone in. DO NOT clone into the same directory you are using as your Eclipse workspace, this causes problems when importing the project later.

  3. Wait for the GitHub client to finish cloning the repository and then open Eclipse.

  4. In Eclipse go to File -> Import

  5. Go to Gradle -> Gradle Project and then press Next.

  6. Browse to the location of the LocommotionCommotion directory and then select Build Model.

  7. If the model builds successfully select the root LocommotionCommotion directory and select Finish.

  8. If the model does not build successfully consult our FAQ.

  9. Congratulations! You have imported the project and can now start working on it.

Project Structure

The project has 5 folders in the Package Explorer.

  • LocommotionCommotion
  • LocommotionCommotion-android
  • LocommotionCommotion-core
  • LocommotionCommotion-desktop
  • LocommotionCommotion-ios

The core project is where the vast majority of the code goes. This builds first and is central to all of the other builds. The android, desktop and ios projects correspond to Android , Desktop and iOS runtime environments. This is how our code can compile to multiple platforms from a central code-base. It is the advantage behind using libGDX.

Running the Project

To run the project you will need to use LocommotionCommotion-desktop. Right click the project in the Package Explorer -> Run As -> Java Application.

This option should also now appear in the drop down next to the Run button in the Eclipse hot-bar.
